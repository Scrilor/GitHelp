GitHelp !


Lexique

Master = Suite d'étapes du projet initial.

Branch = Génère une branche dans la suite d'épate du projet qui vous est destiné afin d'isoler son travail des autres collaborateurs.

Clone = Télécharge tous les fichiers sources.

Commit = Enregistre nos modifications dans le repository local, décrit les changement opérer sur le projet ou une partie.

Repository(Repo) = Dossier qui contient tous les fichiers source et évolue en fonction des modifications apportés au projet.

Index = Zone d'assemblage entre votre répertoire de travail et votre dépôt.

Pull = Mets à jour la version locale du code avec les dernières modifications mises en ligne.

Push = Mets à jour la version online du code avec vos modifications.


Commandes Principales/Généralistes de première approche.

git = Fait appel à l'appliaction Git.

git init = Initialise un dossier comme étant destiné au projet en question.

git config --global user.name "Nom Prénom".

git config --global user.email "prénom.nom@cstb.fr".

Ces deux lignes de commandes permettent ainsi de configurer votre identité qui est essentiel car toutes les validations dans Git utilisent ces informations.

git clone "URL" = Permet de cloner un git déjà créer sur la plateforme.

git status = Montre les fichiers modifié précedemment modifié et à ajouter au commit.

git add "chemin du dossier/fichier en question" = Associe les fichiers en question au Git travaillé. Afin qu'il soit pris en compte lors de l'envoi sur la plateforme.

git commit = Modifie la description du dossier ou ficher modifier sur la plateforme dans l'onglet repository.

git push "URL" = Envoie sur la plateforme les modifications effectuer du projet.

git pull "URL" = Récupère la dernière version du projet à jour.

git branch "Nom de la branche" = Créer une version indépendante du projet pour vous seul afin de tester vos propres idée sur le projet sans en incomber les autres collaborateurs. Il est ensuite possible de fusionner au projet inital cette même branche par l'intermédiaire de :

git merge "Nom de la branche indépendante".


P.S. : Il est possible de tout rédiger par l'intermédiaire du site GitLab. Mais cela rends le travail impossible à réaliser sans accès internet.
