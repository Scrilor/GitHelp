Introduction à Git

Cette courte introduction à pour but de vous faire expérimenter les principaux fonctionnement de GitLab par l'intermédiare de ce site Web. En suivant les indications suivantes vous allez modifiez la page Elus du site et ce pour chacun des collaborateurs.


En ligne de commande via Git SCM

- Commencez par initier un dossier comme étant un repository Git local :

git init

- Téléchargez les fichiers sources:

git clone URL

- Configurez votre identité:

git config --global user.name "NOM Prénom"
git config --global user.email "prenom.nom@cstb.fr"

- Regardez quels fichiers sont à ajouter à l'index et ajoutez les :

git status
git add Dossiers/fichiers

- Créez une nouvelle branche à votre effigie :

git branch Nomdelabranche

- Changez de branche puis effectuer les modifications que vous souhaitez sur la page Les Elus :

git checkout Nom de la branche

- Push la nouvelle branche modifié :

git push --set-upstream Nomdelabranche origin/Nomdelabranche

- Changez de branche Votre branche -> Branche principale (master) :

git checkout master

- Merge votre branche à la branche principale :

git merge Nomdelabranche

- Pushez à nouveau mais cette fois-ci sur la branche principale :

git push


Avec l'interface graphique Sourcetree


- Clonez le Git en question.


- En récûpérant la branche Dev ce qui vous permettra de travailler dessus.

P.S. : Dans un cas concret Dev est utilisé pour travailler de son côté puisque la branche master à plus vocation à être publique rendant la dernière version disponible pour les utilisateurs.


- Avant de travailler toujours faire un pull de la branche pour éviter toutes erreurs superflues. Pour cela cliquez sur Récupérer.


- Réalisez les modifications souhaitées avec votre éditeur de texte.


- Indexez puis Commitez vos changement dans Sourcetree comme-ceci.En vous dirigeant dans "History".



- Pushez vos modifications avec l'icone push "Envoyer" (Si une erreur survient ouvrez alors la ligne de commande et tapez git push --force).


- Dirigez vous alors sur GitLab pour faire fusionner la branche Dev à la branche master sans la supprimer (la façon la plus simple selon moi). Dans "Branches" vous retrouverez les branches crées et l'option "Merge Request" sur la branche Dev qui vous permet de fusionner les changements.

P.S. : Enlevez l'option de supression de branche elle est plus gênante qu'autre chose.


- Merge la branche Dev a la Master.


Félicitation, vous venez de réussir l'introduction à Git. Pour aller plus loin je vous laisse feuilleter le net.
